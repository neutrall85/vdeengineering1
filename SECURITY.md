# Политика безопасности

## Сообщение об уязвимостях

Если вы обнаружили уязвимость в системе безопасности нашего проекта, пожалуйста, сообщите нам об этом.

### Как сообщить об уязвимости

1. **Не создавайте публичные issues** с описанием уязвимости
2. Отправьте письмо на email: security@vdengineering.ru (будет настроено при деплое)
3. Укажите следующую информацию:
   - Тип уязвимости
   - Шаги для воспроизведения
   - Потенциальное воздействие
   - Рекомендации по исправлению (если есть)

### Ожидаемое время ответа

- Первоначальный ответ: в течение 48 часов
- План исправления: в течение 7 дней
- Исправление: в течение 30 дней для критических уязвимостей

## Реализованные меры безопасности

### HTTP-заголовки безопасности

Проект использует следующие защитные заголовки:

- `X-Frame-Options: SAMEORIGIN` — защита от clickjacking
- `X-Content-Type-Options: nosniff` — предотвращение MIME-sniffing
- `X-XSS-Protection: 1; mode=block` — защита от XSS-атак
- `Referrer-Policy: strict-origin-when-cross-origin` — контроль передачи Referer
- `Permissions-Policy: geolocation=(), microphone=(), camera=()` — ограничение доступа к API устройства
- `Content-Security-Policy` — строгая политика безопасности контента

### Content Security Policy (CSP)

Наш CSP включает:
- `default-src 'self'` — ресурсы только с текущего домена
- `script-src 'self' 'unsafe-inline' https://fonts.googleapis.com https://yandex.ru` — скрипты только из доверенных источников
- `style-src 'self' 'unsafe-inline' https://fonts.googleapis.com https://fonts.gstatic.com` — стили из доверенных источников
- `img-src 'self' data: https:` — изображения с текущего домена и HTTPS
- `connect-src 'self' https://api.volga-dnepr-engineering.ru https://yandex.ru` — API запросы только к доверенным эндпоинтам

### Защита форм

- CSRF-токены для всех форм
- Rate limiting (60 секунд между отправками)
- Honeypot-поля для защиты от ботов
- Валидация данных на клиенте и сервере

### Загрузка файлов

- Проверка размера файла (макс. 10MB)
- Проверка расширений файлов
- Проверка MIME-типов
- Допустимые типы: PDF, DOC, DOCX, XLS, XLSX

### Защита данных

- Консольные логи удаляются в production сборке
- Чувствительные данные не хранятся в localStorage
- Использование sessionStorage для временных токенов
- Санитизация HTML перед рендерингом

### Зависимости

- Минимальное количество внешних зависимостей
- Регулярное обновление пакетов
- Использование Dependabot для мониторинга уязвимостей
